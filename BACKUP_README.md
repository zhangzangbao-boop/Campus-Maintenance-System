# 数据备份与恢复功能说明

## 功能概述

系统已实现自动数据备份功能，确保数据安全不丢失。

## 功能特性

1. **自动备份**：每天凌晨2点自动执行一次数据库备份
2. **手动备份**：管理员可以随时手动触发备份
3. **备份列表**：查看所有备份文件及其信息
4. **数据恢复**：从备份文件恢复数据库
5. **自动清理**：自动删除超过保留天数的旧备份（默认30天）

## 配置说明

在 `application.yml` 中可以配置以下参数：

```yaml
backup:
  directory: ./backups  # 备份文件存储目录（相对路径或绝对路径）
  retention-days: 30    # 备份文件保留天数，超过此天数的备份将被自动删除
```

## API 接口

所有备份相关接口都需要管理员权限（ADMIN角色）。

### 1. 手动创建备份

**接口**：`POST /api/admin/backup/create`

**请求**：无需参数

**响应示例**：
```json
{
  "code": 200,
  "message": "备份成功",
  "data": {
    "fileName": "repairdb_backup_20241218_020000.sql",
    "filePath": "F:\\桌面\\CampusSmartMaintenanceCenter-main (1)\\-reportingcenter\\backups\\repairdb_backup_20241218_020000.sql",
    "fileSize": 1234567,
    "backupTime": "2024-12-18T02:00:00",
    "status": "SUCCESS"
  }
}
```

### 2. 获取备份列表

**接口**：`GET /api/admin/backup/list`

**请求**：无需参数

**响应示例**：
```json
{
  "code": 200,
  "message": "获取备份列表成功",
  "data": [
    {
      "fileName": "repairdb_backup_20241218_020000.sql",
      "filePath": "...",
      "fileSize": 1234567,
      "backupTime": "2024-12-18T02:00:00",
      "status": "SUCCESS"
    }
  ]
}
```

### 3. 恢复数据库

**接口**：`POST /api/admin/backup/restore`

**请求体**：
```json
{
  "fileName": "repairdb_backup_20241218_020000.sql"
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "恢复成功"
}
```

**⚠️ 警告**：恢复操作会覆盖当前数据库的所有数据，请谨慎操作！

### 4. 删除备份文件

**接口**：`DELETE /api/admin/backup/{fileName}`

**请求**：路径参数 `fileName` 为备份文件名

**响应示例**：
```json
{
  "code": 200,
  "message": "删除成功"
}
```

## 定时任务

系统会在每天凌晨2点自动执行备份任务。定时任务配置在 `BackupScheduler` 类中：

```java
@Scheduled(cron = "0 0 2 * * ?")  // 每天凌晨2点执行
public void scheduledBackup() {
    // 执行备份
}
```

如果需要修改备份时间，可以修改 cron 表达式：
- `0 0 2 * * ?` - 每天凌晨2点
- `0 0 0 * * ?` - 每天凌晨0点
- `0 0 3 * * ?` - 每天凌晨3点

## 备份文件格式

备份文件命名格式：`repairdb_backup_YYYYMMdd_HHmmss.sql`

例如：`repairdb_backup_20241218_020000.sql`

## 系统要求

1. **MySQL工具**：系统需要安装 MySQL 客户端工具（`mysqldump` 和 `mysql`）
   - Windows：通常随 MySQL 安装，需要添加到系统 PATH
   - Linux/Mac：通过包管理器安装 `mysql-client` 或 `mariadb-client`

2. **权限要求**：
   - 数据库用户需要有备份和恢复的权限
   - 备份目录需要有写入权限

3. **磁盘空间**：确保备份目录有足够的磁盘空间

## 故障排查

### 1. 备份失败

**可能原因**：
- MySQL 客户端工具未安装或未在 PATH 中
- 数据库连接信息错误
- 备份目录没有写入权限

**解决方法**：
- 检查 `mysqldump` 命令是否可用：在命令行执行 `mysqldump --version`
- 检查 `application.yml` 中的数据库配置
- 检查备份目录权限

### 2. 恢复失败

**可能原因**：
- 备份文件不存在或已损坏
- 数据库连接信息错误
- 数据库用户权限不足

**解决方法**：
- 检查备份文件是否存在且完整
- 检查数据库配置和用户权限
- 查看日志文件获取详细错误信息

### 3. 定时任务未执行

**可能原因**：
- 应用未启用定时任务功能
- 服务器时间不正确

**解决方法**：
- 确保主应用类有 `@EnableScheduling` 注解
- 检查服务器系统时间

## 注意事项

1. **备份频率**：建议保持每天一次备份，避免过于频繁影响系统性能
2. **备份存储**：建议将备份文件存储到安全的位置，可以考虑：
   - 网络存储（NAS）
   - 云存储（定期上传）
   - 外部硬盘（定期复制）
3. **恢复测试**：定期测试恢复功能，确保备份文件可用
4. **监控告警**：建议添加备份失败的监控告警，及时发现问题

## 安全建议

1. **备份文件权限**：设置备份目录的访问权限，防止未授权访问
2. **备份加密**：对于敏感数据，可以考虑对备份文件进行加密
3. **异地备份**：定期将备份文件复制到其他位置，防止单点故障
4. **备份验证**：定期验证备份文件的完整性
